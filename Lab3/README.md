## Задание 1

**Дано**

| Параметр | Значение |
|---|---|
| IP-адрес узла | 192.168.200.139 |
| Исходная маска | 255.255.255.0 (/24) |
| Новая маска | 255.255.255.224 (/27) |

**Найти**

| Параметр | Значение |
|---|---|
| Количество бит подсети | 3 |
| Количество созданных подсетей | 8 |
| Количество бит узлов | 5 |
| Количество узлов | 30 |
| Сетевой адрес | 192.168.200.128 |
| Первый узел | 192.168.200.129 |
| Последний узел | 192.168.200.158 |
| Широковещательный адрес | 192.168.200.159 |

**Расчёт через биты**

```
Веса битов:  128 64 32 16 8 4 2 1

Маска /27 = 255.255.255.224
11111111.11111111.11111111.11100000
                            ^^^ 3 бита подсети (от /24 к /27)

Бит подсети:  27 - 24 = 3   ->  подсетей 2^3 = 8
Бит узлов:    32 - 27 = 5   ->  узлов 2^5 - 2 = 30

Разбор 4-го октета (139 = 10001011):
192.168.200. 100 01011
             ^^^ сеть  ^^^^^ хосты

network   (хосты = 0):  100 00000 = 128  ->  192.168.200.128 /27
broadcast (хосты = 1):  100 11111 = 159  ->  192.168.200.159
1st host:  192.168.200.129
last host: 192.168.200.158
```

## Вопрос для повторения

**Почему маска подсети так важна при анализе IPv4-адреса?**

Сам по себе IP-адрес - это просто 32 бита, и без маски невозможно понять, где
заканчивается сетевая часть и начинается хостовая. Именно маска проводит эту
границу: показывает, сколько старших бит относятся к адресу сети, а сколько
младших к хостам.

От этой границы зависит всё: адрес сети, широковещательный адрес, диапазон
доступных узлов и то, находятся ли два устройства в одной подсети (а значит,
нужен ли им маршрутизатор для связи). Один и тот же IP-адрес с разными масками
будет принадлежать совершенно разным сетям.
