## Цель

консольный и удалённый (telnet) доступ, настроить IP-адрес управления и

## 2. Базовая настройка коммутатора
Выполнена конфигурация основных параметров:

```
hostname SW1
enable secret 5 $1$mERr$hx5rVt7rPNoS4wqbXKX7m0
service password-encryption
no ip domain-lookup
banner motd ^CAuthorized access only^C
!
interface Vlan1
 ip address 192.168.1.10 255.255.255.0
ip default-gateway 192.168.1.1
!
line con 0
 password 7 0822455D0A16
 login
 logging synchronous
 exec-timeout 5 0
!
line vty 0 4
 password 7 0822455D0A16
 login
 transport input telnet
```

Настроено:
- имя устройства `SW1`;
- пароль на привилегированный режим (`enable secret`, MD5-хэш);
- `service password-encryption`- шифрование паролей в конфигурации;
- пароль на консольную линию + `login`, `logging synchronous`, `exec-timeout`;
- пароль на линии vty 0-4 + `login`, `transport input telnet`;
- баннер MOTD;
- IP-адрес управления на SVI Vlan1 - 192.168.1.10/24;
- шлюз по умолчанию 192.168.1.1.

## Ответы на вопросы методички

**1. Почему для первоначальной настройки коммутатора используется консольное подключение, а не Telnet или SSH?**

На новом коммутаторе IP-адрес не назначен, а удалённый доступ (vty) не настроен, поэтому подключиться по сети невозможно. Доступ к устройству есть только физически, через консольный порт. Кроме того, консоль не зависит от работоспособности сети и используется для аварийного доступа.

**2. Сколько интерфейсов FastEthernet на коммутаторе 2960?**

24 (Fa0/1 - Fa0/24).

**3. Сколько интерфейсов Gigabit Ethernet на коммутаторе 2960?**

2 (Gig0/1 - Gig0/2).

**4. Каков диапазон значений vty-линий?**

0-15, то есть 16 линий (до 16 одновременных удалённых сессий).

**5. Назначен ли IP-адрес сети VLAN 1 по умолчанию? Включён ли интерфейс?**

По умолчанию IP-адрес не назначен, интерфейс VLAN 1 находится в состоянии administratively down (выключен).

**6. Под управлением какой версии Cisco IOS работает коммутатор? Имя файла образа?**

Версия 15.0(2)SE4, образ `2960-lanbasek9-mz.150-2.SE4.bin`.

**7. Интерфейс по умолчанию включён или выключен? Что нужно сделать, чтобы включить интерфейс?**

По умолчанию неподключённый порт находится в состоянии notconnect. Для включения интерфейса используется команда `no shutdown` в режиме настройки интерфейса.

**8. Для чего нужна команда `login`?**

Команда `login` указывает линии (console или vty) запрашивать пароль при входе. Без неё заданный пароль игнорируется, а подключение либо проходит без запроса, либо отклоняется (если пароль не задан).

**9. Зачем необходимо настраивать пароль VTY для коммутатора?**

Без пароля на линиях vty удалённое подключение по Telnet или SSH будет отклонено (линия с `login`, но без пароля, не пускает). Пароль vty ограничивает удалённый доступ только авторизованными пользователями.

**10. Что нужно сделать, чтобы пароли не хранились в незашифрованном виде?**

Применить команду `service password-encryption`, которая шифрует пароли в конфигурации (слабый Type 7). Для пароля привилегированного режима использовать `enable secret` (стойкий MD5-хэш) вместо `enable password`. Для удалённого доступа предпочтительнее SSH вместо Telnet, так как Telnet передаёт данные в открытом виде.
